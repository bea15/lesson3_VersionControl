Highlight #1 What is Git?

Git is a version control system that is typically seen as the most widely used. The main different between git and the other version control systems is the way git processes data. Most of the other systems store the information as a list of file-based changes. These systems think if the information they keep as a set of files and the changes made to each file over the time period. Git doesn’t think of and store its data like the other version control systems. Git thinks of the data as like sets of screenshots in a file system. Every time you save your project in git, it basically takes a picture of your file.

Highlight #2 History of Git

The Linux kernel is a large open source software project. During most of Linux’s maintenance lifetime, changes in the software were passed around as archived files. Starting in 2002, the Linux kernel project started to use a proprietary DVCS called BitKeeper. In 2005, the community that developed Linux kernel and the commercial company that made BitKeeper broke down and the tools free of charge status was taken away. This forced the community that developed Linux kernel to make their own tools from the tips they learned from BitKeeper. Some of the goals of the new system were speed, simple design, and fully distributed.

Highlight #3 Version Control Systems

Version Control System, or VCS, is a system that records changes to a file or a set of files. For graphic and web designers, if they want to keep every version of a picture or layout, a Version Control System would be good to use. A common version control method that a lot of people use is the Local Version Control because it is easier to use. Even though it is simple to use, it is easy to make errors. It is common to forget which directory you are in and accidently write to the wrong file or copy over files you don’t mean to.

Highlight #4 Getting a Git Repository

There are two main approaches to getting a git repository. The first uses an existing repository or project and imports it into git. The second approach copies an existing git repository from another server. If you are going from an existing project in git, you have to go to the projects directory. This will make a new sub directory called .git that contains all of your needed repository files. At that point nothing in your project has been tracked yet. If you start from version controlling existing files, you should probably stating tracking the files and do an initial commit.

Highlight #5 GitHub Account Setup

GitHub is the largest host for git repository, and the central point for communication and collaboration for many developers and projects. The first step is making an free account for the user by going to https://github.com. You must choose a username that is not already taken, type in you email, create a password, and click the big green “Sign up for GitHub” botton. The next step you will see is pricing options but you can just select the free option. You will need to go to your email and verify it before you start anything else. It is super simple and easily accessible for anyone that wants to create an account.
